Features

Access and Credentials (Complete)

Access and credentials were setup in a professional way to prioritize security and industry standards.

  • Use of Root account avoided where possible
  • AWS Organizations used to manage multiple OUs
  • Different OUs for production and test.
  • AWS SSO (IAM Identity Center) from root to access accounts in OUs

This was done to simulate and gain experience in how professional cloud teams manage their accounts.

S3 and CloudFront Hosting (Complete)

This website is hosted on S3 and distributed with Amazon CloudFront

  • Domain registered with CloudFlare and site stored in private S3 bucket
  • SSL/TLS Certificate generated with AWS Certificate Manager and verified with Cloudflare DNS
  • CloudFront accesses private S3 bucket with REST api and OAC (Origin Access Control) for best practice and security
  • CloudFront reroutes all HTTP traffic to HTTPS for security
  • Cloudflare used to set up appropriate DNS records

Back-End API and Database (Complete)

A Back-End API was implemented with the serverless stack of DynamoDB, Lambda, and API Gateway

  • DynamoDB Database created to store site visitor timestamps
  • Lambda function created in python that when triggered, will append a visitor to the DynamoDB table including the X-Forwarded-For header (sourceip) and timestamp
  • HTTP API created with API Gateway. When called, the API will trigger the Lambda function
  • API is called when anyone enters the website for the first time
  • New REST API created and proxied to DynamoDB table to scan table and retrieve total item count
